# Office 500 Notes

一个用于归档和发布多个独立 HTML 页面的仓库。  
这些页面以静态 HTML 文件的形式存放，不依赖任何后端服务，可直接通过浏览器打开查看。

## 项目简介

本项目的主要目的是集中管理一批独立、自包含的 HTML 文档。  
每个页面都是完整的 HTML 文件，不依赖外部资源（除非在页面内显式引用）。  
适合作为临时记录、信息展示或轻量级存档用途。

## 目录说明

仓库根目录下存放所有 HTML 页面文件，每个文件对应一个独立页面。  
目录结构可能包含子文件夹用于分类或组织，但不会引入复杂的构建系统。

```
.
├── example-page.html
├── notes/
│   ├── note-2024.html
│   └── note-2025.html
├── README.md
└── ...
```

- **根目录**：放置通用的或未分类的 HTML 页面。
- **子文件夹**：按主题、时间或用途分组，方便检索。

## 页面归档说明

所有归档的 HTML 页面均遵循以下原则：

- 每个文件均为完整 HTML，包含必要的 `<html>`, `<head>`, `<body>` 结构。
- 页面内容均为静态文本、样式和脚本，不主动发起网络请求。
- 文件名应具有描述性，便于识别内容。
- 页面内不包含任何商业推广、外部链接跳转或诱导性内容。

## 维护说明

- 仓库维护者会不定期添加新页面或修正已有页面内容。
- 欢迎通过 Issues 提出建议或报告问题，但请勿提交与具体网站推广相关的请求。
- Pull Request 仅接受纯文本或样式修正，不接受涉及外部链接或商业内容的新增页面。
- 本仓库不保证所有页面长期可用，也不提供在线托管服务。

## 使用方式

1. 克隆仓库到本地。
2. 直接双击 HTML 文件，用浏览器打开即可查看。
3. 也可通过任何静态文件服务器托管整个仓库目录。
